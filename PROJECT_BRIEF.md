# 项目策划简报

## 1. 这个面试项目要打动谁

- HR：看见这是一个“不是套模板”的作品，有业务理解，也有表达能力
- 业务负责人：看见你理解国际物流场景，不只是会写页面
- 用人经理：看见你能把 AI、OCR、自动化、Web 界面串成可落地方案

## 2. 从岗位描述里提炼出的核心关键词

### 业务关键词

- 国际物流
- 海运 / 空运 / 铁路 / 快递
- 海外 B 端客户
- 阿里国际站多店铺运营

### 能力关键词

- AI 应用
- OCR
- RPA
- 自动化
- 低代码
- Python
- Prompt Engineering
- Web 基础能力

### 岗位真正要解决的问题

1. 单证处理重复且耗时
提单、箱单、发票需要人工识别、录入、校验。

2. 报价响应慢
价格表分散，客户需求多样，人工匹配效率低。

3. 询价流程碎片化
需要向多个同行或供应商反复询价、回收、汇总、比较。

4. 官网与内部系统割裂
未来希望对接英文官网/API，不只是内部工具。

## 3. 从 Seabay 官网提炼出的重要信息

### 企业定位

- 深圳市西岸国际货运有限公司，2004 年成立
- 经商务部、交通部和海关总署批准的国际货运代理企业
- 官方网站为 `www.seabay.cn`

### 主营业务

- 集装箱海运
- 散货拼箱
- 散杂货运输
- 租船包机
- 国际空运
- 海陆空联运
- 中港运输
- 仓储装卸
- 代理报关 / 代理报检
- FBA 头程物流
- 货运保险
- 代理进出口

### 公司亮点

- 阿里巴巴首批国际物流合作商之一
- WCA 中国区创始会员之一
- 全球有 2000 多家联盟合作伙伴
- 强调 IT 能力和线上工具能力

### 官网最有价值的“数字化资产”

- 三字代码查询
- 空运跟踪
- 海运跟踪
- 海运整柜价格查询
- 空运价格查询
- 港口查询
- 航空公司查询
- 船公司查询

### 一个关键判断

这家公司不是单纯做“货代服务介绍页”，而是长期把“物流服务 + 查询工具 + 内容流量 + 线上线索”结合在一起。

这意味着你的 demo 如果只做普通官网，会偏弱；如果做成“AI 化升级版官网/工作台”，会更对口。

## 4. 最适合做成网站 Demo 的功能

## 方案 A：AI 自动录单 Demo

### 交互形式

- 上传提单 / 箱单 / 发票截图或 PDF
- 页面右侧实时展示 OCR 识别结果
- 自动提取字段：发货人、收货人、起运港、目的港、件毛体、箱型、品名
- 标记缺失字段、可疑字段、冲突字段
- 一键生成“待录入 TMS 订单”

### 为什么适合面试展示

- 最贴岗位职责
- 最能体现 AI + OCR + 业务字段理解
- 可以做得很直观，演示效果强

## 方案 B：智能报价系统 Demo

### 交互形式

- 输入客户需求：运输方式、起运港、目的港、货量、时效、是否清关/派送
- 上传或预置几份价格表
- 系统自动匹配报价规则
- 输出 3 档推荐方案：低价 / 均衡 / 时效优先
- 同时生成英文报价摘要

### 为什么适合面试展示

- 非常符合“未来对接英文官网”的描述
- 能体现你对结构化数据、规则引擎、提示词和前端表单的理解
- 比纯 OCR 更像产品

## 方案 C：供应商询价机器人 Demo

### 交互形式

- 创建询价任务
- 选择目标供应商或航线池
- 页面模拟机器人批量发送询价
- 收到不同供应商回复后自动归一化
- 生成比价表、推荐结论和风险提示

### 为什么适合面试展示

- 很贴近岗位中的 RPA / OpenClaw / 龙虾机器人方向
- 能展示“流程自动化”而不只是“单个 AI 调用”
- 容易做成时间线式可视化，观感好

## 方案 D：物流查询助手 Demo

### 交互形式

- 一个 AI 助手入口
- 可查询港口、机场三字代码、航司、船司、轨迹、价格线索
- 根据客户提问自动给出下一步建议

### 为什么适合面试展示

- 能呼应官网已有工具体系
- 适合作为包装层，但不建议单独作为主 demo

## 5. 最推荐的成品形态

不要做成单功能页面，建议做成：

“国际物流 AI 自动化展示站”

其中包含：

1. 一个高级感首页
体现企业业务、全球覆盖、数字化能力、AI 升级方向。

2. 三个可交互 demo
- AI 自动录单
- 智能报价
- 机器人询价

3. 一个轻量数据看板
显示节省工时、录入准确率、报价时效、询价覆盖率等指标。

## 6. 推荐 MVP

如果你想尽快做出可展示版本，优先做下面这一版：

### 首页包装

- Hero
- 主营业务卡片
- 官方工具能力映射
- AI 自动化价值说明

### Demo 1

- AI 自动录单

### Demo 2

- 智能报价

### Demo 3

- 机器人询价流程模拟

这样已经足够支撑一次面试演示。

## 7. 建议你在面试时怎么讲

可以把这个项目讲成一句话：

“我没有把它做成普通企业官网，而是把官网能力、国际物流业务流程和 AI 自动化岗位职责结合起来，做成了一个可交互的业务 demo，用来展示 AI 在单证处理、报价和询价三个核心环节的落地方式。”

## 8. 我对你当前最推荐的方向

优先级建议如下：

1. 先做“官网首页 + 3 个 AI demo 入口”
2. 再把“自动录单”和“智能报价”做成交互最完整的两块
3. 最后补“机器人询价”的流程演示和数据看板

## 9. 下一轮讨论最值得定下来的 3 件事

1. 这个 demo 更偏“官网展示感”还是更偏“产品工作台”
2. 首页视觉风格要走哪条路线
3. 第一版先做静态假数据交互，还是直接接简单本地 mock API
