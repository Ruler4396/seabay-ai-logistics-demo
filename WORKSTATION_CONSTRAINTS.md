# Workstation UI Constraints

## Product framing

- Default to `internal workstation`, not landing page, not storytelling page.
- Prioritize `state`, `actions`, `queue`, `results`.
- Keep the enterprise website link as a secondary external reference only.

## Content density

- Prefer short labels over explanatory paragraphs.
- One panel should answer one question: `what is pending`, `what is selected`, `what can I do next`.
- Empty states must be one short sentence whenever possible.
- Avoid “why this matches”, “how to present”, “interview angle”, or similar meta sections in the UI.

## Copy rules

- Use operational wording: `queue`, `task`, `status`, `selected`, `ready`, `blocked`, `run`, `print`.
- Avoid promotional phrasing, self-congratulation, or narrative framing.
- Avoid repeated explanation of the same context across pages.
- If a section cannot drive an action or show status, it should usually be removed.

## Layout rules

- Prefer dashboard, worklist, side summary, data table, and action bar patterns.
- Keep top-level navigation compact and tool-like.
- Reserve large hero areas for route, shipment, or workload status, not explanation.
- Keep visual polish in spacing, hierarchy, and motion; not in decorative copy blocks.

## Future edits

- Before adding any new section, ask: does this improve execution, monitoring, or decision making?
- If the answer is no, do not add it.
